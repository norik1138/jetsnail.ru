Есть некая транспортная организация “Реактивная Улитка”.
В организации есть автопарк состоящий из разного вида транспорта - есть легковые авто, есть грузовики и фургоны.
Так же есть различная спецтехника - возможно в будущем будет приобретаться что то еще.
У каждой единице техники есть свой водитель который за нее ответственен.
Еще каждая единица может находиться в определенном статусе (активная, на ремонте, продана , не используется)

Нужно создать мини систему для учета всей этой техники
 для того чтобы владелец Бизнеса, Василий, мог зайти и быстро посмотреть перечень своей техники и в каком она состоянии.

* Необходимо развернуть последнюю версию Laravel
* Инициализировать git
* Сделать первый коммит на github

1. Нужна возможность зарегистрироваться и войти в систему (Ларавел работает из коробки)
2. CRUD - для водителей (Полностью Лaравел)
3. CRUD - для транспорта (Полностью на VUE)
4. Остальной CRUD на твое усмотрение
5. Некая общая главная страница с карточками авто которая которая видна из веб тоесть не закрытая системой аутентификации

Задействовать хотя бы поверхностно следующие подходы:

* однофайловые компоненты
* axios
* vue-router
* можно чуть чуть vuex
* заюзать Laravel Passport для аутентификации фронтенд из части VUE. Если даже не получится либо не хватит времени то хотя бы изучить теорию так как на собеседовании будут вопросы по нему

* Сделать финальный коммит на github

Из гитхаб я заливаю ваш проект сразу на боевой сервер для того что-бы вживую его оценить прямо в открытом вебе
